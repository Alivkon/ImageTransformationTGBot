# ImageTransformationBot — Telegram-бот для AI-генерации фото

Коммерческий Telegram-бот, позволяющий пользователям трансформировать свои фотографии с помощью нейросети: сохраняет лицо, но меняет позу, одежду или фон по текстовому описанию.

---

## Стек технологий

| Область | Инструменты |
|---|---|
| Язык | Python 3.11+ |
| Фреймворк | aiogram 3.x (async) |
| БД | SQLite + aiosqlite |
| HTTP | aiohttp |
| Оплата | ЮKassa (Telegram Payments API) |
| AI-генерация | KIE.ai REST API (модель `google/nano-banana`) |
| Конфигурация | python-dotenv |
| Деплой | Docker + Docker Compose |

---

## Ключевые возможности

- **AI-трансформация фото** — пользователь отправляет фото с текстовым промптом, бот возвращает перегенерированное изображение
- **Платёжная система** — интеграция с ЮKassa через Telegram Payments, поддержка 54-ФЗ (онлайн-чек)
- **Баланс и монетизация** — внутренний кошелёк, пополнение на суммы 100–1000 ₽, 1 бесплатная генерация для новых пользователей
- **Надёжная обработка ошибок** — при сбое генерации средства автоматически возвращаются на баланс
- **Админ-уведомления** — middleware оповещает администратора о каждом платеже в реальном времени
- **Логирование** — структурированные логи на каждом этапе (запрос, результат, ошибки)

## Архитектура

```
ritualTGBot/
├── bot.py              # Точка входа, инициализация Dispatcher
├── config.py           # Конфигурация через env-переменные
├── database.py         # Слой данных (SQLite, async CRUD)
├── handlers/
│   ├── start.py        # /start, регистрация пользователя
│   ├── generate.py     # Приём фото, запуск генерации, отправка результата
│   └── payment.py      # Пополнение баланса, обработка успешных платежей
├── services/
│   └── nanobanana.py   # Клиент KIE.ai API (создание задачи + polling статуса)
├── keyboards/
│   └── inline.py       # Inline-клавиатуры
└── middlewares/
    └── admin_notify.py # Middleware уведомлений администратора
```

Проект придерживается **слоистой архитектуры**: хэндлеры обрабатывают входящие события, сервисный слой инкапсулирует работу с внешним API, слой данных изолирует все операции с БД.

## Технические решения

### Асинхронная архитектура
Весь стек построен на `asyncio`: aiogram, aiosqlite, aiohttp — без блокирующих вызовов.

### Polling внешнего API
Генерация изображений занимает несколько секунд. Реализован неблокирующий polling с настраиваемым интервалом и максимальным количеством попыток (таймаут 3 минуты).

### Атомарность операций с балансом
Списание средств происходит до запроса к API. При любой ошибке (сетевой или от AI-сервиса) баланс восстанавливается — пользователь не теряет деньги.

### База данных
Три таблицы: `users`, `payments`, `generations`. Поддерживается живая миграция схемы (добавление столбцов без пересоздания таблиц).

## Запуск

### Docker (рекомендуется)

```bash
# 1. Создать .env на основе примера
cp .env.example .env
# Заполнить BOT_TOKEN, KIE_API_KEY, YOOKASSA_TOKEN, ADMIN_ID

# 2. Собрать и запустить
docker compose up -d
```

База данных хранится в папке `data/` — она монтируется как volume и не теряется при пересборке образа.

Остановить: `docker compose down`  
Логи: `docker compose logs -f`

### Локально

```bash
pip install -r requirements.txt
cp .env.example .env
python bot.py
```

### Переменные окружения

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен бота от @BotFather |
| `KIE_API_KEY` | API-ключ KIE.ai |
| `YOOKASSA_TOKEN` | Токен провайдера ЮKassa |
| `ADMIN_ID` | Telegram ID администратора |
| `GENERATION_COST` | Стоимость одной генерации в ₽ (по умолчанию 20) |
| `MIN_TOPUP` | Минимальная сумма пополнения (по умолчанию 100) |
| `FREE_GENERATIONS` | Бесплатных генераций для новых пользователей (по умолчанию 1) |
